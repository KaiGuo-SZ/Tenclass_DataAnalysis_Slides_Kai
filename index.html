VLOOKUP - 范围查找

按名单 / ID 快速提取信息
使用场景
  • 从大量数据里按名单 / ID，快速提取对应信息
  • 例如：ID 查订单、订单号查购课时间
公式语法
=VLOOKUP(查找值, 查找范围, 列序数, 0)
(查找值, table, col, FALSE)
参数说明
参数说明
查找值你要找谁（如：姓名 "张伟"、体验课名称 "AA朗诵"）
查找范围在哪一片区域里找（查找值必须在区域第一列）
列序数你要返回的数据，在查找范围里是第几列
匹配类型0/FALSE = 精确匹配；1/TRUE = 模糊匹配（常用 0）

常见问题 1：姓名不一致

报名表“张伟” vs 订单表“张伟 ”（带空格）

解决方案：TRIM 函数

TRIM 是 Excel 中的文本函数，作用就是去掉单元格内容前后的空格，但保留文字之间的单个空格。

公式示例：

=TRIM(" 张伟 ") → 结果是 "张伟"
VLOOKUP + TRIM 组合用法
=VLOOKUP(TRIM(A2), 订单表!$B$2:$D$10, 3, 0)
常见问题 2：VLOOKUP 限制
  • 只能从左往右查、#N/A 不友好

XLOOKUP 登场

Excel 2021+ / Office 365 推荐

推荐使用 XLOOKUP（Excel 2021 以上 / Office 365）
=XLOOKUP(查找值, 查找列, 返回列, [未找到时显示的内容])

对比 VLOOKUP：

功能VLOOKUPXLOOKUP
查找方向只能从左往右左右任意方向
列数计算需要数第几列直接选返回列
反向查找不支持支持
模糊匹配需要写参数默认精确匹配
找不到提示固定 #N/A可自定义，如“未转化”
小练习

用XLOOKUP的方法，统计体验课“AA朗诵”学员购买正价课的下单时间，让找不到时显示“无订单”。

=XLOOKUP(TRIM(A2), 订单表!$B$2:$B$10, 订单表!$D$2:$D$10, "无订单")

常见错误 · 绝对引用

复制/下拉公式时最常踩坑

下拉公式时范围为什么会跑偏？

  • 错误：=VLOOKUP(A2, A1:E5, 5, 0) → 下拉变成 A2:A6、A3:A7
  • 正确：=VLOOKUP(A2, $A$1:$E$5, 5, 0) → 按 F4 锁定
小提示

分渠道分析转化率时经常需要复制公式到多列，绝对引用尤其重要。
但有时需要混合引用（锁列不锁行），比如对比不同周的转化率。

COUNTIF 基础（策略场景）

单条件计数：先把条件写对

COUNTIF：单条件计数

公式：=COUNTIF(计数范围, 条件)

示例：

=COUNTIF(ROI列, ">=0.8")          ' 统计ROI达标班长数（预期：3人）
=COUNTIF(大组列, "社群一部")       ' 统计社群一部班长数（预期：3人）

常见错误：

  • 条件必须用引号：">=0.8" 不能写成 >=0.8
  • 文本条件："社群一部" 不能写成 社群一部

COUNTIFS 多条件（策略场景）

所有条件同时满足
进阶任务（COUNTIFS）

统计“社群一部”中 ROI >= 0.8 的班长数

任务解读
  • 条件1：大组 = 社群一部
  • 条件2：ROI >= 0.8

COUNTIFS：所有条件同时满足

公式：=COUNTIFS(范围1, 条件1, 范围2, 条件2, ...)

示例：

=COUNTIFS(大组列, "社群一部", ROI列, ">=0.8")

→ 预期结果：2（张伟 0.85，刘洋 0.81）

情境 · 条件判断（业务方向）

活跃度分级：高 / 中 / 低

场景：你轮岗到「业务」团队

需求：根据体验课学员的总学习时长（分钟）和到播次数，将学员按照活跃度分为高 / 中 / 低三级，并决定是否“重点跟进”。

规则

  • 时长 > 60 或 到播次数 > 1 → 中（重点跟进：是）

IF 基础（业务场景）

二选一判断 → 多级容易乱

IF：二选一判断

公式：=IF(条件, 满足时返回, 不满足时返回)

二选一判断：及格/不及格、合格/不合格、是/否

案例

用 IF 判断是否重点跟进（是/否），先用简单条件（时长>60）。

=IF(时长>60, "是", "否")

嵌套 IF 实现多级（但易乱）

=IF(AND(学习时长>120, 到播次数>3), "高", IF(OR(学习时长>60, 到播次数>1), "中", "低"))

缺点：括号多了易错，条件顺序敏感。

推荐使用 IFS（下一页）。

IFS 多条件（业务场景）

从严格到宽松：先高后中

IFS：清晰的多条件判断

公式：=IFS(条件1, 结果1, 条件2, 结果2, 条件3, 结果3, ...)

按顺序判断：满足第一个条件就返回第一个结果，依次检查

示例：

=IFS(
   AND(学习时长>120, 到播次数>3), "高",
   OR(学习时长>60, 到播次数>1), "中",
   TRUE, "低"
)
  • 最后一个 TRUE 表示“其他情况”，必须写，否则返回 #N/A
  • 条件顺序必须从严格到宽松（先高后中）

重点跟进列：

=IF(OR(等级="高", 等级="中"), "是", "否")

TEXTJOIN + IF 实现条件合并

将“未交作业”的学员筛选出来，再用顿号拼成一段可直接发送的名单

TEXTJOIN 基础

把多个文本合并成一个

TEXTJOIN：把多个文本合并成一个

语法：=TEXTJOIN("、", TRUE, 文本区域)

简单示例：

=TEXTJOIN("、", TRUE, A2:A10)   ' 将A2到A10的名字用顿号连起来

情境实操：将“未完成作业”的学员用顿号合并

=TEXTJOIN("、", TRUE, IF(作业状态列="未交", 姓名列, ""))

进阶：按班级分别生成

=TEXTJOIN("、", TRUE, IF((班级="朗诵1班")*(作业状态="未交"), 姓名, ""))

→ 结果：李娜、陈静

AI 时代，为什么还要学数据分析？

从“工具执行”到“人机协作”

提问

当 AI 已经能做很多事
当 AI 已经能：
  • 写代码
  • 做图表
  • 跑模型
  • 自动生成报告
为什么还需要数据分析？

当工具变强时，人的价值在哪里？

核心观点

能力模型升级

AI 改变的，不是“数据分析是否重要”

而是：

数据分析师的能力模型

从“会做”转向“会定义问题 + 会判断 + 会推动落地”。

旧能力 vs 新能力
过去（工具执行）现在（人机协作）
记函数、背语法清晰描述业务问题
手动清洗数据设计分析框架
做固定图表判断结果是否合理
写重复代码追问原因与假设

AI 的定位

强助手，不是决策者
AI 更像：

一个会写代码、做图表、跑模型的分析助手

AI 能帮你：
  • ✅ 清洗数据
  • ✅ 写代码
  • ✅ 做分析
  • ✅ 找异常
  • ✅ 生成假设
但 AI 不知道：
  • 真实业务背景
  • 数据口径问题
  • 哪个问题最重要
  • 哪个结论能落地

AI 如何辅助数据分析？

先把场景说清楚
场景

某在线教育业务：

  • 近 3 个月销售额持续下滑
  • 已有销售数据 + 用户活跃数据
目标

找出下滑原因，并提出优化方向

错误 vs 正确提问

把问题问到“可执行”
❌ 错误问法

“帮我分析销售为什么下降。”

问题：

  • 没有业务背景
  • 没有字段说明
  • 没有分析目标
✅ 更好的问法

我有：

  • 销售表（成本、产值、转化人数等）
  • 活跃表（DAU、学习时长等）

近 3 个月销售额持续下滑。

请帮我：

  1. 拆解可能原因
  2. 设计分析框架
  3. 列出关键指标

AI 如何协助分析

拆解 + 执行
Step 1：拆解问题

AI 帮你分析：

  • 是流量下降？
  • 是转化率下降？
  • 是 ROI 变差？
  • 还是用户活跃下降？
Step 2：执行分析

AI 可以：

  • 自动计算指标
  • 做趋势分析
  • 找异常班级
  • 生成图表与代码

AI 如何协助决策

假设 + 判断
Step 3：生成假设

AI 可能发现：

  • 获客成本上涨 → ROI 下滑
  • 学习时长下降 → 转化降低
  • 某些班级表现异常
Step 4：人的判断

真正重要的是：

  • 哪个问题最值得解决？
  • 是否需要补充新数据？
  • 最终应该调整什么业务策略？

数据分析通用 Prompt 模板

问得好，才做得对

一个好的 AI 提问，通常包含：

要素清单
  1. 业务背景：发生了什么问题？
  2. 数据说明：有哪些表？有哪些字段？
  3. 分析目标：想找什么原因？验证什么假设？
  4. 输出要求：需要代码、图表、结论，还是建议？
提示

把“要你做什么”说清楚，把“你有什么数据”说清楚。
